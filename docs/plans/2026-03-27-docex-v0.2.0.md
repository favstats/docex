# docex v0.2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Add 10 new features, 7 robustness improvements, and npm publish readiness.

**Architecture:** All new features follow the existing pattern: static class methods taking a Workspace object, exposed via the DocexEngine fluent API and CLI.

---

## New Features (10 tasks)

### Task 1: doc.diff(other.docx)
Compare two documents. Output tracked changes showing what changed between them.
- `doc.diff("revised.docx")` returns a new document with w:ins/w:del markup
- Useful for R&R: compare submitted version vs revised version
- Algorithm: paragraph-level text diff, then word-level within changed paragraphs
- **Files:** Create `src/diff.js`, add `diff()` to docex.js, add CLI `docex diff a.docx b.docx --output diff.docx`

### Task 2: doc.accept() / doc.reject()
Accept or reject tracked changes programmatically.
- `doc.accept()` accepts ALL tracked changes (removes w:del elements, unwraps w:ins)
- `doc.accept(id)` accepts a specific change by ID
- `doc.reject()` rejects ALL (removes w:ins, unwraps w:del)
- `doc.reject(id)` rejects specific change
- `doc.revisions()` lists all tracked changes with IDs
- **Files:** Create `src/revisions.js`, add to docex.js + CLI

### Task 3: doc.toc()
Generate or update a table of contents.
- Scans headings, builds TOC with page refs (using field codes)
- Inserts after title / before first heading
- **Files:** Add to `src/paragraphs.js` or new `src/toc.js`

### Task 4: doc.footnote(anchor, text)
Insert footnotes anchored to specific text.
- Creates footnote in word/footnotes.xml
- Adds footnote reference at anchor position in document.xml
- Manages footnote IDs and relationships
- **Files:** Create `src/footnotes.js`, add to docex.js + CLI

### Task 5: doc.bibliography(bibFile, options)
Generate formatted reference list from .bib file.
- Parse BibTeX, format in APA/Chicago/custom style
- Insert as paragraphs before or after "References" heading
- Does NOT replace Zotero; alternative for users without Zotero
- options.style = "apa" | "chicago" | "harvard"
- **Files:** Create `src/bibliography.js`, add to docex.js + CLI

### Task 6: doc.highlight(text, color)
Highlight text with a background color.
- Adds `<w:highlight w:val="yellow"/>` to the run's rPr
- Colors: yellow, green, cyan, magenta, red, blue, darkYellow
- Tracked by default (wraps in w:rPrChange)
- **Files:** Add to `src/paragraphs.js`

### Task 7: doc.style(preset)
Apply a full document style preset.
- Presets: "academic" (Times New Roman 12pt, double-spaced, justified, 1in margins)
- Presets: "polcomm" (Political Communication journal format)
- Presets: "joc" (Journal of Communication format)
- Modifies styles.xml + document defaults
- **Files:** Create `src/presets.js` with journal-specific configs

### Task 8: doc.merge(other.docx)
Merge another document into the current one.
- Appends all paragraphs from other.docx after current content (before sectPr)
- Handles image relationship conflicts (renumber rIds)
- Handles style conflicts (keep current document styles)
- **Files:** Create `src/merge.js`

### Task 9: doc.wordCount()
Accurate word count excluding headers, captions, footnotes.
- Returns { total, body, headings, captions, footnotes, abstract }
- Splits by section for per-section counts
- **Files:** Add to `src/paragraphs.js`

### Task 10: doc.metadata({title, author, keywords})
Set document properties (docProps/core.xml).
- title, author, subject, keywords, description, created, modified
- Creates docProps/core.xml if missing
- **Files:** Create `src/metadata.js`

## Robustness Improvements (7 tasks)

### Task 11: Better cross-run editing (mixed-state TextMap)
Port docx-editor's mixed-state handling for text spanning w:ins boundaries.
- When replacing text that partially overlaps a tracked insertion, correctly
  handle the three cases: regular text, inside-insertion text, mixed
- **Files:** Rewrite `src/textmap.js` and update `src/paragraphs.js`

### Task 12: Comment anchoring on exact phrases
Currently comments anchor to the whole paragraph. Fix to anchor to the exact run(s) containing the anchor text.
- commentRangeStart goes before the specific run, not the paragraph
- commentRangeEnd goes after the specific run, not before </w:p>
- **Files:** Update `src/comments.js`

### Task 13: Format-preserving replace across runs
When text spans bold + plain runs, preserve the formatting boundary in the replacement.
- Port docx-editor's approach: emit one w:ins run per original formatting span
- **Files:** Update `src/paragraphs.js`

### Task 14: Property-based fuzz testing
Generate random operations on the test fixture and verify invariants.
- Random: replace random substring, insert at random heading, add comment at random text
- Invariants: paragraph count >= original, valid zip, file size within 50%
- 100 random iterations per test run
- **Files:** Create `test/fuzz.test.js`

### Task 15: npm publish readiness
- Add .npmignore (exclude test/, docs/, reference/)
- Add prepublish script that runs tests
- Verify package size is reasonable (<100KB without reference)
- Add CI badge placeholder in README
- **Files:** Update package.json, create .npmignore

### Task 16: Compact OOXML cheat sheet
Extract the 200 most essential lines from the 2,967-line reference into a compact cheat sheet.
- Focus on: tracked changes XML, comment XML, image XML, table XML
- Copy-paste-ready snippets
- Small enough to include in a Claude Code skill without context bloat
- **Files:** Create `reference/ooxml-cheatsheet.md`, update skill to reference it

### Task 17: Better error messages with closest matches
When text is not found, show the 3 closest matching paragraphs.
- Use Levenshtein distance or substring overlap scoring
- Print: "Text not found. Did you mean: ..." with the 3 best matches
- **Files:** Update `src/paragraphs.js`, `src/comments.js`
