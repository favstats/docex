## v0.4.1: AI Agent Reliability

Features that make docex bulletproof for automated/AI-driven editing.

| Feature | API | What it does |
|---------|-----|-------------|
| Snapshot/rollback | `doc.snapshot()` / `doc.rollback()` | Save document state in memory. If an operation goes wrong, roll back without touching disk. Like git stash but in-memory. |
| Assert | `doc.assert(paraId, "expected text")` | Verify a paragraph contains expected text before operating. Fails fast instead of silently editing the wrong paragraph. |
| Diff summary | `doc.diffSummary("other.docx")` | "3 paragraphs changed, 1 added, 2 comments added" without full diff. For agents deciding whether to look closer. |
| Operation receipts | Every mutation returns `{ success, paraId, matched, context }` | Agent can verify each operation worked without re-reading the whole document. |

**Files:** Modify `src/docex.js`, `src/workspace.js`, `src/paragraphs.js`, `src/diff.js`
**Tests:** `test/agent-reliability.test.js`
**Estimated:** ~4 new methods, ~15 tests

## v0.4.2: Writing Process Tools

Features for restructuring, splitting, and manipulating document structure.

| Feature | API | What it does |
|---------|-----|-------------|
| Outline | `doc.outline()` | Extract headings as flat list for restructuring overview. |
| Move section | `doc.moveSection("Discussion", { before: "Results" })` | Reorder sections by moving all paragraphs between headings. |
| Split document | `doc.split("Methods")` | Split at a heading into two .docx files. For submitting methods as supplement. |
| Extract abstract | `doc.extractAbstract()` | Pull abstract as plain text. For conference submission forms that want it in a text box. |
| Duplicate section | `doc.duplicateSection("Results", "Results (Appendix)")` | Copy a section with a new heading. |
| Append section | `doc.section("Methods").append("New paragraph")` | Add content at end of a specific section. |

**Files:** Create `src/sections.js`, modify `src/docex.js`
**Tests:** `test/sections.test.js`
**Estimated:** ~6 new methods, ~20 tests

## v0.4.3: Collaboration and Privacy

Features for multi-author workflows and data protection.

| Feature | API | What it does |
|---------|-----|-------------|
| Redact | `doc.redact("participant names", "[REDACTED]")` | Replace sensitive text with placeholder, log mapping for later restoration. For participant data, not author names (that's anonymize). |
| Unredact | `doc.unredact()` | Restore redacted text from the logged mapping. |
| Section permissions | `doc.permissions({ Methods: "readonly" })` | Mark sections as read-only using Word's document protection at section level. |
| Compare styles | `doc.compareStyles("polcomm")` | Preview what would change: "12 paragraphs change font, margins narrow by 0.5in". Without applying. |

**Files:** Create `src/redact.js`, modify `src/submission.js`, `src/presets.js`
**Tests:** `test/collaboration.test.js`
**Estimated:** ~4 new methods, ~12 tests

## v0.4.4: Quality and Writing Checks

Features for catching errors and improving writing quality before submission.

| Feature | API | What it does |
|---------|-----|-------------|
| Spell check | `doc.spellcheck()` | Flag words not in bundled dictionary. Returns `[{ word, paraId, suggestions }]`. No external service. |
| Consistency check | `doc.consistencyCheck()` | Find inconsistencies: "268,635" vs "268635", same entity spelled differently, number format mismatches. |
| Passive voice | `doc.passiveVoice()` | Flag sentences in passive voice. Returns `[{ sentence, paraId, suggestion }]`. |
| Sentence length | `doc.sentenceLength({ max: 40 })` | Flag sentences over N words. Returns `[{ sentence, paraId, wordCount }]`. |
| Readability | `doc.readability()` | Flesch-Kincaid score, average sentence length, vocabulary complexity. |

**Files:** Create `src/quality.js`, include a bundled English word list (~50K common words as a .json or generated array)
**Tests:** `test/quality.test.js`
**Estimated:** ~5 new methods, ~20 tests

## v0.4.9: Provenance and Changelog

Features for tracking document history and verifying integrity.

| Feature | API | What it does |
|---------|-----|-------------|
| Changelog | `doc.changelog()` | Embedded changelog inside the .docx as a custom XML part. Every docex operation logged with timestamp, author, description. Survives file moves and renames. |
| Changelog query | `doc.changelog.since("2026-03-18")` | Filter changelog entries by date. |
| Changelog on save | Automatic | Every `doc.save()` appends operations to the changelog. No opt-in needed. |
| Origin | `doc.origin()` | Returns: "Created by docex v0.3.0 on 2026-03-27 from template polcomm. 47 operations across 3 sessions." Reads from the embedded changelog. |
| Certify | `doc.certify("submitted to Political Communication")` | Embed a SHA-256 hash of the document content + a label + timestamp as a custom XML part. Immutable record of "this was the state at this moment." |
| Verify certification | `doc.verifyCertification()` | Check if document content still matches the last certification hash. Returns `{ certified: true, label: "submitted to...", date: "..." }` or `{ certified: false, changedSince: "2026-03-27" }`. |
| Certification history | `doc.certifications()` | List all certification points: `[{ label, date, hash, stillValid }]`. |

Implementation: uses `customXml/` parts inside the .docx zip (standard OOXML extension mechanism). Three custom XML parts:
- `docex-changelog.xml` -- array of operation entries
- `docex-origin.xml` -- creation metadata
- `docex-certifications.xml` -- array of hash checkpoints

**Files:** Create `src/provenance.js`
**Tests:** `test/provenance.test.js`
**Estimated:** ~7 methods, ~20 tests
